# 产品定位和功能边界

> 本文件定义产品的目标、核心功能和边界，防止功能蔓延。

## 产品定位

- **一句话描述**：DocChat 是面向独立开发者/小团队的文档智能客服 SaaS 解决方案，帮助用户零代码嵌入文档聊天组件，并让 RAG 检索质量可迭代
- **目标用户**：独立开发者 / 小团队 — 产品有 FAQ 需求但不想自己搭 RAG
- **核心价值**：① 零代码嵌入 — 上传文档就能生成聊天组件，5 分钟上线；② RAG 可控可测 — 评测集 + 版本管理，让检索质量可迭代

### 角色定义

| 角色 | 说明 |
|------|------|
| 租户管理员 | 注册账号、创建租户工作空间、上传文档、管理 API Key、查看用量、配置聊天组件、邀请成员 |
| 团队成员 | 查看和管理知识库文档、配置评测集、版本回滚、调整聊天组件外观 |
| 终端访客 | 在嵌入的聊天组件中提问，获得基于文档的回答 |

## 核心功能

| # | 功能模块 | 描述 | 优先级 | 状态 |
|---|---------|------|--------|------|
| 1 | 租户与用户管理 | 注册登录、租户工作空间、团队成员邀请、角色权限（管理员/成员/只读） | P0 | 待开发 |
| 2 | 知识库管理 | 文档上传（PDF/Markdown/文本）、自动切分向量化、文档列表查看、版本记录与回滚、删除已索引数据 | P0 | 待开发 |
| 3 | 异步任务处理 | 大文档上传后后台切分+向量化，前端轮询任务状态，支持任务进度查看和失败重试 | P0 | 待开发 |
| 4 | RAG 对话服务 | 基于知识库的问答对话，返回带来源引用的回答，对话即焚不持久化 | P0 | 待开发 |
| 5 | 聊天组件嵌入 | 生成可嵌入的 JS 脚本，类似 Intercom 的轻量级聊天 widget，支持外观配置（品牌色、欢迎语、图标） | P0 | 待开发 |
| 6 | API Key 与访问控制 | API Key 生成/吊销、每日调用次数硬限制 | P1 | 待开发 |
| 7 | 用量统计 | 调用量、Token 消耗、对话数统计，按租户维度汇总 | P1 | 待开发 |
| 8 | 评测集 | 内置问答对管理、自动评测检索 Hit Rate、评测结果历史对比 | P1 | 待开发 |
| 9 | 聊天组件预览与模拟测试 | 管理后台组件配置页内嵌预览窗口：① 实时预览外观配置效果（品牌色/欢迎语/图标修改后立刻反映）；② 模拟终端访客问答，输入问题获得真实 RAG 回答（对话不持久化，不计入用量统计）；③ 刷新按钮一键重置预览窗口到初始状态 | P1 | 待开发 |
| 10 | 聊天组件高级配置 | 自定义主题、多语言界面、对话引导语配置 | P2 | 待规划 |

## 功能边界（不做）

- ❌ 实时计费/账单/支付集成 — MVP 只做调用次数硬限制，不做金额计算和支付
- ❌ 多语言 UI — 目标用户是独立开发者/小团队，中文界面即可
- ❌ 对话历史持久化与导出 — 终端访客的对话是临时性的，问完即焚
- ❌ 多知识库/多 Bot — 每个租户一个知识库一个 Bot，不做多实例
- ❌ SSO/OAuth 第三方登录 — 目标用户是小团队，邮箱注册即可
- ❌ 文档协作编辑 — 产品定位是"上传+生成"，不是文档编辑器
- ❌ Webhook/事件回调 — MVP 无集成需求，后续按需加
- ❌ 移动端适配 — 管理后台桌面端优先，聊天组件本身响应式即可

## 非功能性要求

| 维度 | 要求 |
|------|------|
| 性能 | 对话 API < 2s（P95），管理后台 API < 500ms（P95） |
| 可用性 | 单实例部署，不要求高可用 |
| 安全 | 多租户严格隔离（TenantContext + AOP 自动过滤，Milvus 按租户独立 collection）+ API Key 鉴权 + JWT Token + 输入校验 + 文件上传多层校验 |
| 国际化 | 中文界面，不做 i18n |
| 数据量 | 单租户文档 < 100 个，单文档 < 50MB，对话并发 < 100 QPS |
| 可测试性 | 核心 Service 单元测试覆盖率 > 80%，关键流程集成测试 |
| 可观测性 | SLF4J + Logback，Request-Id 链路追踪，结构化日志，日志脱敏 |
| 部署 | Docker Compose（MVP），后续按需上云 |

## 里程碑

| 阶段 | 目标 | 预计时间 |
|------|------|----------|
| MVP | 租户管理 + 知识库管理 + 异步任务处理 + RAG 对话服务 + 聊天组件嵌入（P0 全部交付） | |

| V2 | 聊天组件高级配置（P2），根据用户反馈决定后续方向 | 按需 |
