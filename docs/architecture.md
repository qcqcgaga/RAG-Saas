# DocChat 架构设计

## 系统架构

DocChat 采用 Monorepo 结构，包含三个子系统：

```
┌────────────────────────────────────────────────────────────┐
│                    docchat-web (Vue 3)                      │
│                 管理后台 — Ant Design Vue                   │
├────────────────────────────────────────────────────────────┤
│                  docchat-server (Spring Boot)               │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐      │
│  │  tenant  │ │knowledge │ │   task   │ │   chat   │      │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘      │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐                   │
│  │  widget  │ │   stat   │ │  apikey  │                   │
│  └──────────┘ └──────────┘ └──────────┘                   │
├────────────────────────────────────────────────────────────┤
│  ┌──────────┐  ┌──────────┐  ┌──────────┐                 │
│  │PostgreSQL│  │  Milvus  │  │  Redis   │                 │
│  └──────────┘  └──────────┘  └──────────┘                 │
├────────────────────────────────────────────────────────────┤
│               @docchat/chat-widget (IIFE JS)                │
│                嵌入式聊天组件 — 任意网页可嵌入               │
└────────────────────────────────────────────────────────────┘
```

## 后端模块说明

| 模块 | 职责 | 阶段 |
|------|------|------|
| module-tenant | 注册登录、租户工作空间、角色权限 | MVP |
| module-knowledge | 文档上传/删除、切分向量化、版本管理 | MVP |
| module-task | 异步任务提交、状态查询、失败重试 | MVP |
| module-chat | RAG 问答、向量检索、LLM 调用 | MVP |
| module-widget | 聊天组件配置、JS 脚本生成 | MVP |
| module-stat | API 用量统计、Token 消耗统计 | V1 |
| module-apikey | API Key 生成/吊销、调用次数限制 | V1 |
| module-eval | 评测集管理、检索质量评测 | V1 |

## 多租户隔离方案

- PostgreSQL：通过 Hibernate Filter + TenantContext 自动注入 `tenant_id` 条件
- Milvus：每个租户独立的 Collection
- Redis：Key 前缀包含 `tenant:{id}:`
- 文件存储：按 `tenant/{id}/` 分目录
