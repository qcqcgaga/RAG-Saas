# 约束文件变更建议

> 项目：DocChat — 文档智能客服 SaaS
> 版本：MVP 0.1.0
> 制定日期：2026-06-25

## 变更建议清单

### 1. codeRule.md 变更

#### 新增规则：敏感配置默认值安全要求

**建议内容**：
```markdown
### 安全配置默认值规则

- JWT_SECRET 默认值必须 >= 32 字节（满足 HMAC-SHA256 最低要求）
- 数据库密码等敏感配置默认值不得为空
- 生产环境必须通过环境变量覆盖所有敏感默认值
- 默认值应包含明显的"非生产"标识（如 `dev-only-xxx`）
```

**变更原因**：MVP部署时发现JWT_SECRET默认值`changeme-in-production`只有22字节，不满足HMAC-SHA256 >= 32字节要求，导致Spring Boot启动失败。

**优先级**：P1

---

### 2. tech.md 变更

#### 新增策略：人工测试环境要求

**建议内容**：
```markdown
### 人工测试环境要求

- 人工测试必须在 Docker Compose 完整环境中执行
- 禁止在 H2 内存数据库环境中进行人工测试
- 人工测试前必须确保 PostgreSQL + Redis + Milvus 全部健康
- 人工测试前必须调用 /user-manual 刷新用户手册
```

**变更原因**：MVP的集成测试全部在H2环境中执行，与真实环境差异巨大，导致部署时才发现5个环境差异问题。

**优先级**：P0

---

### 3. product.md 变更

#### 调整：P0功能占位实现的处理策略

**建议内容**：
```markdown
### P0功能占位实现策略

当前 P0 功能中有 2 项使用占位实现：
1. Embedding → 随机向量
2. LLM → 模拟流式

这两项占位使得 RAG 对话核心功能无法真正使用。
策略：
- MVP 阶段允许占位实现，但必须在上线验证报告中明确标注
- V1 版本必须替换为真实 API，否则不允许进入 V1 发布流程
```

**变更原因**：当前P0功能有占位实现但未在PRD中明确标注，导致交付时对"是否满足P0验收标准"产生歧义。

**优先级**：P2
