2. 在登录页面，点击切换到 **注册** 标签页

4. 点击 **注册** 按钮
5. 注册成功后，观察页面是否自动跳转到知识库页面

#### 预期结果

### US-003: 邀请团队成员 (P0) — R2修复回归

1. 登录后进入 **租户设置** 页面

4. 点击 **确定** 完成邀请
5. **退出当前账号**（清除 localStorage 或使用隐私窗口）
6. 用被邀请人的邮箱和刚才设定的密码登录

#### 预期结果

- 邀请成功，新成员出现在列表中
- 邀请弹窗中有"初始密码"输入框
- 被邀请人使用管理员设定的密码**可以成功登录**
- 登录后能看到租户数据

### US-011: 访客在聊天组件中提问 (P0) — 占位功能验证

**作为** 终端访客，**我想要** 在聊天组件中提问，**以便** 快速获得答案

> ⚠️ MVP阶段：Embedding使用随机向量占位，LLM使用模拟流式占位。
> 本轮验证重点是：API能否正常返回流式响应、来源引用格式是否正确。

#### 测试步骤

1. 在聊天组件页面，查看嵌入脚本中的 **widget_token**
2. 使用 widget_token 发起对话：
   ```bash
   curl -X POST http://localhost:8080/api/v1/chat/conversations \
     -H "Authorization: Bearer <widget_token>" \
     -H "Content-Type: application/json" \
     -d '{"question": "这个产品是什么？"}'
   ```
3. 或在聊天组件页面预览中提问

#### 预期结果

- API返回SSE流式响应（非500/400错误）
- 响应中包含来源引用格式
- 对话不持久化

#### 实际结果

- [ ] ✅ 通过 / ❌ 未通过
- 问题描述：我需要在预览界面直接测试，先确认确实可以进行问答
