### US-001: 注册账号并创建租户工作空间 (P0)

**作为** 租户管理员，**我想要** 注册账号并创建租户工作空间，**以便** 开始使用服务

#### 测试步骤

1. 打开浏览器访问 **http://localhost**
2. 在登录页面，点击切换到 **注册** 标签页
3. 填写注册信息：
   - **邮箱**：输入一个有效的邮箱地址（如 `testuser001@example.com`）
   - **密码**：输入至少8位且包含大写字母、小写字母和数字的密码（如 `Test1234`）
   - **租户名称**：输入工作空间名称（如 `测试团队`）
4. 点击 **注册** 按钮

#### 预期结果

- 注册成功，系统自动登录
- 页面跳转到管理后台（知识库页面）
- 页面顶部显示租户名称
- 系统自动创建了默认知识库

#### 实际结果

- [❌] ✅ 通过 / ❌ 未通过
- 问题描述：整体OK，但是刚注册的账号这个接口会报错
http://localhost/api/v1/knowledge/documents?page=1&size=20
请求方法
GET
状态代码
500 Internal Server Error
{
    "code": 50000,
    "msg": "系统繁忙，请稍后重试"
}

### US-011: 访客在聊天组件中提问 (P0)

**作为** 终端访客，**我想要** 在聊天组件中提问，**以便** 快速获得答案

#### 测试步骤

> ⚠️ 此测试需要先确保知识库中有已完成处理的文档

1. 使用 widget_token 直接访问对话 API：
   ```bash
   curl -X POST http://localhost:8080/api/v1/chat/conversations \
     -H "Authorization: Bearer <widget_token>" \
     -H "Content-Type: application/json" \
     -d '{"question": "这个产品是什么？"}'
   ```
2. 或在聊天组件页面预览中提问

#### 预期结果

- 系统返回 SSE 流式响应
- 回答附带来源引用（文档名+段落）
- 对话不持久化

#### 实际结果

- [❌] ✅ 通过 / ❌ 未通过
- 问题描述：______阻塞_____

> **重要提示**：当前 Embedding 使用随机向量占位，LLM 使用模拟流式占位。
> 因此此用户故事的真实效果需要等接入真实 API 后才能验证。
> MVP 阶段验证重点是：API 是否能正常返回流式响应、来源引用格式是否正确。

## 端到端流程测试

### E2E-01: 完整用户旅程（注册→上传→配置→对话）

这是最关键的测试场景，验证用户从注册到使用完整流程是否畅通。

#### 测试步骤

1. **注册**：在 http://localhost 注册新账号
2. **登录**：自动登录后进入管理后台
3. **上传文档**：在知识库页面上传一个 Markdown 文件
4. **等待处理**：等待文档处理完成（可能需要几秒到几分钟）
5. **查看任务**：在任务页面查看文档切分任务状态
6. **配置组件**：在聊天组件页面配置品牌色和欢迎语
7. **获取嵌入脚本**：复制嵌入脚本代码
8. **发起对话**：使用 widget_token 发起一次对话

#### 预期结果

- 全流程畅通，无阻塞报错
- 每一步都有合理的界面反馈
- 用户体验无明显障碍

#### 实际结果

- [✅] ✅ 通过 / ❌ 未通过
