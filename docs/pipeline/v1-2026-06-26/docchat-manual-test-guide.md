# 人工测试引导文档

> 项目：DocChat — 文档智能客服 SaaS
> 版本：v1-2026-06-26
> 测试日期：2026-06-26
> 服务地址：http://localhost:8080（后端）/ http://localhost:5173（前端）

## 测试前准备

1. 确认服务已启动（参考用户手册"快速开始"章节）
   - Docker Compose: `docker compose up -d` 或
   - 本地启动: 后端 `mvn spring-boot:run`，前端 `pnpm dev`
2. 准备测试数据：至少一个已注册租户 + 一个已创建知识库 + 至少一份已处理完成的文档
3. 打开浏览器访问前端地址 http://localhost:5173
4. 使用管理员账号登录

## 用户故事测试场景

### US-V1-001: 生成 API Key (P1)

**作为** 租户管理员，**我想要** 生成 API Key，**以便** 让聊天组件通过 Key 鉴权访问对话服务

#### 测试步骤
1. 登录管理后台
2. 在左侧导航栏点击 **API Key 管理**
3. 点击 **创建 API Key** 按钮
4. 在弹窗中输入名称（如"生产环境Key"），点击确认
5. 查看生成的完整 Key 值

#### 预期结果
- Key 以 `dc_` 前缀开头
- Key 仅在创建时展示完整值，弹窗关闭后无法再次查看完整值
- 列表中显示脱敏值（如 `dc_a1b2c****3f4g`）
- Key 状态为"有效"
