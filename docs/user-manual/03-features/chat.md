# RAG 对话

## 功能概述

RAG（Retrieval-Augmented Generation）对话是 DocChat 的智能问答核心。终端访客在聊天组件中提问时，系统会：

1. 将问题转换为向量
2. 在 Milvus 中检索最相关的知识片段
3. 将检索结果和问题一起发送给大语言模型
4. 流式生成回答，并附带来源引用

## 对话体验

### 访客操作流程

1. 访客打开嵌入了 DocChat 聊天组件的网页
2. 点击页面右下角的聊天图标，打开聊天窗口
3. 在输入框中输入问题
4. 按 **Enter** 或点击 **发送** 按钮
5. 系统流式输出回答，实时显示生成过程
6. 回答完成后，底部显示来源引用

[截图：聊天组件对话界面]

### 回答特点

| 特性 | 说明 |
|------|------|
| 流式输出 | 回答逐字流式显示，无需等待完整生成 |
| 来源引用 | 每条回答附带引用的文档名称和段落位置 |
| 上下文理解 | 基于知识库内容回答，非通用闲聊 |
| 对话即焚 | 访客对话不持久化存储，关闭窗口后消失 |

### 来源引用

回答下方会显示 **来源引用** 区域，列出回答所依据的文档片段：

```
来源引用
├── 产品使用手册.pdf (第3段)
├── API文档.md (第7段)
```

每个引用包含：
- **文档名称**：回答引用的来源文档
- **段落序号**：文档中具体被引用的片段位置

## 技术细节

### 流式传输（SSE）

对话使用 Server-Sent Events（SSE）实现流式输出，API 响应格式为 `text/event-stream`。

每个 SSE 事件包含以下类型：

| 事件类型 | 说明 |
|----------|------|
| `token` | 生成的文本片段，逐步累加 |
| `done` | 生成完成，附带来源引用列表 |
| `error` | 发生错误，返回错误信息 |

### 鉴权方式

- **聊天组件**使用 `widget_token` 鉴权，通过 `Authorization: Bearer <token>` 请求头传递
- **管理后台**使用 JWT Token 鉴权，与聊天组件鉴权方式不同

### 对话即焚

DocChat 遵循"对话即焚"原则：
- 终端访客的对话**不会持久化存储**到数据库
- 对话仅在生成过程中存在于服务端内存
- 访客关闭聊天窗口后，对话内容不可恢复
- 这保护了访客隐私，也降低了数据合规风险

## 性能指标

| 指标 | 目标 |
|------|------|
| 对话 API 响应时间 | < 2s（P95） |
| 首个 token 输出 | < 1s |
| 并发对话支持 | < 100 QPS |

## 限制说明

- 每次对话独立，不支持多轮上下文连续对话（MVP 阶段）
- 回答质量取决于知识库文档的完整性和切分效果
- 如遇到回答不准确，建议优化文档内容的结构化程度
