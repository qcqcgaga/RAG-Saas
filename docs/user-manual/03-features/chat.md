# RAG 对话

## 功能概述

RAG（Retrieval-Augmented Generation）对话是 DocChat 的智能问答核心。终端访客在聊天组件中提问时，系统会：

1. 将问题转换为向量
2. 在 Milvus 中检索最相关的知识片段
3. 将检索结果和问题一起发送给大语言模型
4. 流式生成回答，并附带来源引用

## 对话体验

### 访客操作流程

1. 访客打开嵌入了 DocChat 聊天组件的网页
2. 点击页面右下角的聊天图标，打开聊天窗口
3. 在输入框中输入问题
4. 按 **Enter** 或点击 **发送** 按钮
5. 系统流式输出回答，实时显示生成过程
6. 回答完成后，底部显示来源引用

[截图：聊天组件对话界面]

### 回答特点

| 特性 | 说明 |
|------|------|
| 流式输出 | 回答逐字流式显示，无需等待完整生成 |
| 来源引用 | 每条回答附带引用的文档名称和段落位置 |
| 上下文理解 | 基于知识库内容回答，非通用闲聊 |
| 对话即焚 | 访客对话不持久化存储，关闭窗口后消失 |

### 来源引用

回答下方会显示 **来源引用** 区域，列出回答所依据的文档片段：

```
来源引用
├── 产品使用手册.pdf (第3段)
├── API文档.md (第7段)
```

每个引用包含：
- **文档名称**：回答引用的来源文档
- **段落序号**：文档中具体被引用的片段位置

## 双鉴权模式（V1 新增）

V1 版本对话接口支持两种鉴权方式，系统根据 Token 前缀自动判断：

| Token 前缀 | 鉴权方式 | 场景 | 统计 | 限额 |
|------------|----------|------|------|------|
| `dc_` | API Key | 正式访客对话 | 计入用量统计 | 受每日限额约束 |
| `eyJ` | JWT | 管理后台预览对话 | 不计入统计 | 不受限额约束 |

### 鉴权流程

```
请求到达 /api/v1/chat/conversations
        ↓
提取 Authorization: Bearer <token>
        ↓
判断 token 前缀
   ├── dc_ → API Key 鉴权 → 校验 Key → 检查限额 → 计入统计
   └── eyJ → JWT 鉴权 → 解析 tenantId → 不计入统计
```

### API Key 鉴权细节

1. 提取 API Key，计算 SHA-256 哈希
2. 校验 Key 是否有效且未吊销
3. 检查租户每日调用限额
4. 递增限额计数器
5. 设置 AuthContext（供统计 AOP 使用）

如果 API Key 无效或已吊销，返回 `40802 - API Key 无效或已吊销`。
如果每日调用次数超限，返回 `40804 - 每日调用次数超限`。

详见 [API Key 管理](apikey.md)。

## 技术细节

### 流式传输（SSE）

对话使用 Server-Sent Events（SSE）实现流式输出，API 响应格式为 `text/event-stream`。

每个 SSE 事件包含以下类型：

| 事件类型 | 说明 |
|----------|------|
| `token` | 生成的文本片段，逐步累加 |
| `done` | 生成完成，附带来源引用列表 |
| `error` | 发生错误，返回错误信息 |

### LLM 配置

V1 版本支持租户级 LLM 配置，对话时使用的 LLM 取决于配置：

- 已配置租户 LLM → 使用租户自定义的 API 地址和模型
- 未配置 → 使用系统默认 LLM

详见 [LLM 配置](llm-config.md)。

### 对话即焚

DocChat 遵循"对话即焚"原则：
- 终端访客的对话**不会持久化存储**到数据库
- 对话仅在生成过程中存在于服务端内存
- 访客关闭聊天窗口后，对话内容不可恢复
- 这保护了访客隐私，也降低了数据合规风险
- 预览对话和管理后台测试对话同样遵循此原则

## 性能指标

| 指标 | 目标 |
|------|------|
| 对话 API 响应时间 | < 2s（P95） |
| 首个 token 输出 | < 1s |
| 并发对话支持 | < 100 QPS |

## 限制说明

- 每次对话独立，不支持多轮上下文连续对话
- 回答质量取决于知识库文档的完整性和切分效果
- 如遇到回答不准确，建议优化文档内容的结构化程度
- 使用评测集功能可量化评估检索质量，详见 [评测集管理](eval.md)
- 每日对话调用次数受 API Key 限额约束，超出后对话请求将被拒绝
