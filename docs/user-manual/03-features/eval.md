# 评测集管理

> V1 新增功能

## 功能概述

评测集管理模块让 RAG 检索质量可量化、可迭代。您可以创建评测集、添加问答对、执行评测获取 Hit Rate 指标，并通过历史结果对比验证优化效果。

### 什么是 Hit Rate

Hit Rate（命中率）是衡量 RAG 检索质量的核心指标，表示系统检索到的 Top-K 结果中包含正确答案文档的比例。

```
Hit Rate = 命中题数 / 总题数

其中"命中"指：检索返回的 Top-K 文档中包含该题期望的文档
```

例如：评测集有 10 道题，其中 8 道题的期望文档出现在检索结果中，则 Hit Rate = 80%。

## 创建评测集

### 操作步骤

1. 登录管理后台，进入 **评测集** 页面

[截图：评测集页面 - 空状态]

2. 点击 **创建评测集** 按钮
3. 填写以下信息：
   - **名称**：评测集的标识名称，如"产品FAQ评测v1"
   - **描述**：评测集的说明，如"覆盖产品核心功能的常见问题"
4. 点击 **确认创建**

[截图：创建评测集弹窗]

### 评测集限制

| 限制项 | 说明 |
|--------|------|
| 问答对上限 | 每个评测集最多 **50 对** 问答 |
| 评测集数量 | 每租户可创建多个评测集，无硬性上限 |

## 管理问答对

### 添加问答对

1. 在评测集列表中，点击目标评测集进入详情
2. 点击 **添加问答对** 按钮
3. 填写以下信息：
   - **问题（question）**：模拟终端访客的提问，如"如何重置密码？"
   - **期望文档（expectedDocument）**：期望检索命中的文档名称，如"用户手册.pdf"
4. 点击 **确认添加**

[截图：添加问答对弹窗]

### 查看问答对列表

1. 进入评测集详情
2. 问答对列表显示：
   - **序号**：问答对编号
   - **问题**：模拟提问内容
   - **期望文档**：期望命中的文档名称
   - **操作**：编辑 / 删除

[截图：问答对列表]

### 删除问答对

1. 在问答对列表中找到目标条目
2. 点击 **删除** 按钮
3. 确认删除

> **注意**：删除问答对后，基于该问答对的评测结果中对应条目也会被移除。

## 批量导入

### 操作步骤

1. 在评测集详情页面，点击 **批量导入** 按钮
2. 按照模板格式准备导入数据

[截图：批量导入弹窗]

### 导入格式

批量导入使用 JSON 格式，结构如下：

```json
{
  "pairs": [
    {
      "question": "如何重置密码？",
      "expectedDocument": "用户手册.pdf"
    },
    {
      "question": "支持哪些支付方式？",
      "expectedDocument": "FAQ.md"
    },
    {
      "question": "API 调用频率限制是多少？",
      "expectedDocument": "API文档.pdf"
    }
  ]
}
```

### 导入结果

导入完成后，系统返回导入结果：

| 结果项 | 说明 |
|--------|------|
| 成功数 | 成功导入的问答对数量 |
| 失败数 | 导入失败的条目数量 |
| 失败原因 | 每条失败记录的具体原因 |

> **注意**：导入后如果问答对总数超过 50 对上限，超出部分将被跳过。

## 执行评测

### 操作步骤

1. 在评测集详情页面，点击 **执行评测** 按钮
2. 系统开始执行评测，对每个问答对：
   - 将问题发送给 RAG 检索服务
   - 检查检索结果中是否包含期望文档
   - 记录命中情况
3. 评测完成后，显示结果

[截图：执行评测按钮和进度]

### 评测过程

```
评测集问答对 → 逐条发送问题到 RAG 检索 → 检查 Top-K 结果 → 记录命中/未命中 → 计算 Hit Rate
```

### 评测结果

评测完成后显示以下信息：

| 数据项 | 说明 |
|--------|------|
| Hit Rate | 命中率百分比 |
| 总题数 | 参与评测的问答对数量 |
| 命中题数 | 检索结果包含期望文档的题数 |
| 未命中题数 | 检索结果未包含期望文档的题数 |
| 评测时间 | 评测执行的时间 |

[截图：评测结果概览]

### 评测详情

点击评测结果可查看每道题的详细情况：

| 列 | 说明 |
|----|------|
| 问题 | 评测问题 |
| 期望文档 | 期望命中的文档名称 |
| 实际检索结果 | RAG 返回的 Top-K 文档列表 |
| 是否命中 | 命中 / 未命中 |

[截图：评测详情页面]

## 结果对比

### 查看历史结果

1. 在评测集详情页面，切换到 **评测历史** 标签
2. 查看该评测集的所有历史评测结果

[截图：评测历史列表]

### 对比不同版本

通过历史结果对比，可以验证优化效果：

1. 优化知识库文档或切分参数
2. 重新执行同一评测集
3. 对比前后 Hit Rate 变化

| 评测时间 | Hit Rate | 总题数 | 命中题数 |
|----------|----------|--------|----------|
| 2026-06-20 | 65% | 20 | 13 |
| 2026-06-25 | 80% | 20 | 16 |

> **建议**：每次优化知识库后执行一次评测，记录 Hit Rate 变化，形成优化闭环。

## 评测集管理

### 编辑评测集

1. 在评测集列表中，点击目标评测集的 **编辑** 按钮
2. 修改名称或描述
3. 点击 **保存**

### 删除评测集

1. 在评测集列表中，点击目标评测集的 **删除** 按钮
2. 确认删除

> **重要**：删除评测集会同时删除所有问答对和评测历史结果，该操作不可撤销。

## 优化工作流

评测集的核心价值在于形成"评测 → 优化 → 再评测"的闭环：

```
1. 创建评测集，添加覆盖核心场景的问答对
        ↓
2. 执行评测，获取初始 Hit Rate
        ↓
3. 分析未命中的问题，定位原因
   - 文档缺失 → 补充文档
   - 切分不当 → 调整 chunkSize/overlap
   - 语义偏差 → 优化文档表述
        ↓
4. 优化后重新执行评测
        ↓
5. 对比 Hit Rate 变化，验证优化效果
        ↓
6. 重复 3-5，持续迭代
```

## 注意事项

- 评测执行需要一定时间，取决于问答对数量和 RAG 服务响应速度
- 评测使用当前知识库状态，如果文档有变更，建议重新评测
- Hit Rate 仅衡量检索质量，不衡量生成质量（回答的准确性）
- 期望文档名称需与知识库中的文档名称完全一致
- 评测过程中产生的对话不计入用量统计
