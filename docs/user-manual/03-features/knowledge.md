# 知识库管理

## 功能概述

知识库是 DocChat 的核心功能，用于存储和管理您的产品文档。系统支持上传 PDF、Markdown、纯文本格式的文档，自动进行文本提取、智能切分和向量化，为 RAG 智能对话提供知识支撑。

每个租户拥有一个知识库，知识库中的文档经过处理后即可用于对话问答。

## 上传文档

### 操作步骤

1. 登录管理后台，进入 **知识库** 页面
2. 点击 **上传文档** 按钮
3. 选择文件或拖拽文件到上传区域

[截图：文档上传界面]

4. 可选配置切分参数：
   - **切分策略（strategy）**：文本切分方式，默认自动选择
   - **切分大小（chunkSize）**：每个知识片段的最大字符数
   - **重叠长度（overlap）**：相邻片段的重叠字符数，保证上下文连续性
5. 点击 **确认上传**

### 文件要求

| 项目 | 要求 |
|------|------|
| 支持格式 | PDF（.pdf）、Markdown（.md）、纯文本（.txt） |
| 文件大小 | 单个文件最大 **50MB** |
| 文件名 | 上传后自动使用 UUID 重命名存储，避免冲突 |
| 安全校验 | 文件类型白名单校验 + 文件大小检查 |

### 自动处理流程

上传成功后，系统自动执行以下处理：

```
文件上传 → 文本提取 → 智能切分 → 向量化存储 → 可用于对话
    ↓           ↓           ↓           ↓
  存储文件   提取纯文本  按语义分段  写入 Milvus
```

1. **文本提取**：从 PDF 等格式中提取纯文本内容
2. **智能切分**：按语义段落将长文档切分为知识片段（chunk）
3. **向量化**：将片段转换为向量并写入 Milvus 向量数据库

> **提示**：大文档的处理需要一定时间，请耐心等待。处理期间可以继续上传其他文档。

## 文档管理

### 查看文档列表

1. 进入 **知识库** 页面
2. 文档列表显示以下信息：
   - **标题**：文档名称
   - **文件类型**：PDF / MD / TXT
   - **文件大小**：文档体积
   - **版本**：当前版本号
   - **状态**：处理状态（见下方状态说明）
   - **上传时间**：文档上传时间

[截图：文档列表界面]

### 搜索文档

在文档列表上方的搜索框中输入关键词，按文档标题筛选。

### 按状态筛选

点击状态筛选下拉框，可按以下状态过滤：

| 状态 | 说明 |
|------|------|
| 待处理（pending） | 刚上传，等待处理 |
| 处理中（processing） | 正在进行文本提取和切分 |
| 已完成（completed） | 向量化完成，可用于对话 |
| 失败（failed） | 处理异常，可查看错误信息并重试 |

### 查看文档详情

点击文档列表中的文档标题，查看详细信息：
- 文档基本信息（标题、文件名、类型、大小）
- 当前版本号和状态
- 上传时间和更新时间

### 删除文档

1. 在文档列表中找到目标文档
2. 点击 **删除** 按钮
3. 确认删除操作

> **重要**：删除文档会同步清理 Milvus 中的向量数据，该操作不可撤销。删除后，对话将无法引用该文档内容。

## 版本管理

### 查看版本历史

1. 在文档详情中点击 **版本历史**
2. 查看该文档的所有历史版本

[截图：版本历史界面]

每个版本记录以下信息：
- **版本号**：递增的整数
- **切分片段数**：该版本切分后产生的知识片段数量
- **Milvus 状态**：向量数据同步状态
- **创建时间**：版本生成时间

### 版本回滚

1. 在版本历史中，选择要回滚到的目标版本
2. 点击 **回滚** 按钮
3. 确认回滚操作

回滚后，系统会将 Milvus 中的向量数据恢复到目标版本的状态。

> **注意**：回滚操作会替换当前版本的向量数据，但不会删除版本历史记录。

## 异步任务

文档上传和版本回滚都会创建异步任务。您可以在 **任务** 页面查看任务详情。

### 查看任务列表

1. 进入 **任务** 页面
2. 查看所有任务的列表，包含：
   - **任务类型**：如"文档切分"、"版本回滚"
   - **关联 ID**：关联的文档 ID
   - **状态**：pending / processing / completed / failed
   - **进度**：0-100 的百分比
   - **错误信息**：失败时显示原因

[截图：任务列表界面]

### 重试失败任务

1. 在任务列表中找到失败的任务
2. 点击 **重试** 按钮
3. 系统重新执行该任务

> **提示**：重试前建议检查错误信息，确认问题已解决（如文件格式问题）后再重试。

## 与评测集联动

V1 新增评测集功能后，知识库文档的检索质量可以通过评测集量化验证。建议工作流：

1. 上传文档并等待处理完成
2. 在评测集中创建针对该文档的问答对
3. 执行评测，获取 Hit Rate
4. 根据评测结果优化文档结构或切分参数
5. 重新评测，对比历史结果

详见 [评测集管理](eval.md)。

## 注意事项

- 文档内容会按语义段落自动切分，建议编写文档时使用清晰的标题层级，以获得更好的切分效果
- 删除文档后，相关对话可能无法引用该文档内容，请谨慎操作
- 大量文档同时上传时，处理需要排队，请耐心等待
- 单个租户建议文档数量不超过 100 个，单文档不超过 50MB
