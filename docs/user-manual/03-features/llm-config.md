# LLM 配置

> V1 新增功能

## 功能概述

LLM 配置模块允许租户自定义大语言模型的 API 地址和密钥，灵活适配不同的模型供应商。未配置时，系统使用默认的 LLM 服务，确保开箱即用。

### 为什么需要自定义 LLM 配置

- **成本控制**：使用自己采购的 LLM API，成本自主可控
- **模型选择**：根据业务需求选择不同模型（如 GPT-4、Claude、通义千问等）
- **合规要求**：部分场景要求使用特定供应商的模型服务
- **服务稳定性**：避免对单一供应商的依赖

## 配置说明

### 操作步骤

1. 登录管理后台，进入 **LLM 配置** 页面

[截图：LLM 配置页面 - 未配置状态]

2. 填写以下配置信息：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| API 地址（apiUrl） | LLM 服务的 API 端点 | `https://api.openai.com/v1/chat/completions` |
| API 密钥（apiKey） | 访问 LLM 服务的密钥 | `sk-xxxxxxxxxxxx` |
| 模型名称（modelName） | 使用的模型标识 | `gpt-4o` |

[截图：LLM 配置 - 填写表单]

3. 点击 **测试连通性** 验证配置
4. 测试通过后，点击 **保存配置**

> **安全提示**：API 密钥在界面中脱敏显示（如 `sk-ab****xyz`），系统加密存储。

### 配置状态

| 状态值 | 说明 |
|--------|------|
| 0 | 未配置（使用系统默认 LLM） |
| 1 | 已配置（使用租户自定义 LLM） |

## 测试连通性

### 操作步骤

1. 在 LLM 配置页面填写 API 地址、密钥和模型名称
2. 点击 **测试连通性** 按钮
3. 系统向配置的 API 地址发送测试请求
4. 查看测试结果

[截图：连通性测试结果 - 成功]

### 测试结果

| 结果 | 说明 |
|------|------|
| 连通成功 | 配置正确，LLM 服务可用 |
| 连通失败 | 请检查 API 地址、密钥和模型名称是否正确 |

### 连通失败常见原因

| 原因 | 解决方法 |
|------|----------|
| API 地址错误 | 确认地址格式正确，包含完整路径 |
| API 密钥无效 | 检查密钥是否正确，是否已过期 |
| 模型名称错误 | 确认模型标识与供应商文档一致 |
| 网络不通 | 检查服务器能否访问 LLM API 地址 |
| 余额不足 | 检查 LLM 供应商账户余额 |

## 管理配置

### 查看当前配置

1. 进入 **LLM 配置** 页面
2. 查看当前配置信息：
   - API 地址
   - API 密钥（脱敏显示）
   - 模型名称
   - 配置状态
   - 最后更新时间

[截图：LLM 配置 - 已配置状态]

### 修改配置

1. 在 LLM 配置页面修改需要变更的项
2. 点击 **测试连通性** 验证新配置
3. 测试通过后，点击 **保存配置**

> **注意**：修改 LLM 配置后，所有新的对话请求将使用新配置。正在进行的对话不受影响。

### 删除配置（恢复系统默认）

1. 在 LLM 配置页面，点击 **删除配置** 按钮
2. 确认删除操作
3. 系统恢复使用默认 LLM 服务

[截图：删除配置确认弹窗]

> **注意**：删除配置后，对话将使用系统默认的 LLM 服务。如果系统默认 LLM 不可用，对话将无法正常工作。

## 系统默认 LLM

当租户未配置自定义 LLM 时，系统使用默认 LLM 服务：

| 配置项 | 说明 |
|--------|------|
| 默认供应商 | 讯飞 Coding Plan API |
| 模型 | 系统预设模型 |
| 密钥 | 系统级密钥，由平台管理员配置 |

系统默认 LLM 的特点：
- 开箱即用，无需额外配置
- 所有未自定义配置的租户共享
- 服务可用性由平台保障

## API 兼容性

### 支持的 API 格式

DocChat 的 LLM 配置兼容 OpenAI Chat Completions API 格式。以下供应商的 API 可直接使用：

| 供应商 | API 地址示例 | 说明 |
|--------|-------------|------|
| OpenAI | `https://api.openai.com/v1/chat/completions` | GPT-4、GPT-3.5 等 |
| Azure OpenAI | `https://your-resource.openai.azure.com/...` | 需配置部署名称 |
| 通义千问 | `https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions` | 兼容模式 |
| DeepSeek | `https://api.deepseek.com/v1/chat/completions` | DeepSeek Chat |
| 讯飞星火 | 按讯飞文档配置 | Coding Plan API |
| 其他 | 需兼容 OpenAI 格式 | 支持 OpenAI 兼容接口的服务 |

### 配置要点

1. **API 地址**必须是完整的 Chat Completions 端点路径
2. **API 密钥**需具有对应模型的调用权限
3. **模型名称**需与供应商的模型标识完全一致

## 安全说明

| 安全措施 | 说明 |
|----------|------|
| 密钥脱敏 | 界面和 API 响应中密钥脱敏显示（仅显示前4后4位） |
| 加密存储 | API 密钥在数据库中加密存储 |
| 租户隔离 | LLM 配置严格按租户隔离，不同租户配置互不可见 |
| 权限控制 | 仅管理员可操作 LLM 配置 |

## 注意事项

- 配置 LLM 前务必测试连通性，避免因配置错误导致对话服务不可用
- 不同模型的 Token 计价不同，请关注用量统计中的 Token 消耗数据
- API 密钥变更后请立即测试，确保对话服务正常
- 删除 LLM 配置是恢复系统默认的最快方式，如自定义配置出现问题可先删除再重新配置
