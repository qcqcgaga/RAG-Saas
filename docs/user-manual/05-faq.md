# 常见问题

## 通用问题

**Q: DocChat 支持哪些文档格式？**

A: 目前支持三种格式：PDF（.pdf）、Markdown（.md）和纯文本（.txt）。单文件大小限制为 50MB。

---

**Q: 上传文档后多久可以使用？**

A: 取决于文档大小和当前处理队列。通常小文档（< 1MB）在数秒内完成处理，大文档可能需要数分钟。您可以在任务页面查看处理进度。

---

**Q: 一个租户可以上传多少文档？**

A: 建议单个租户文档数量不超过 100 个，单文档不超过 50MB。如需更大容量，请联系我们。

---

**Q: 对话记录会保存吗？**

A: 不会。DocChat 遵循"对话即焚"原则，终端访客的对话不持久化存储，关闭聊天窗口后对话内容不可恢复。管理后台的预览对话同样不持久化。

---

**Q: 支持多轮对话吗？**

A: 当前版本不支持多轮上下文连续对话，每次提问独立处理。多轮对话将在后续版本中支持。

---

**Q: 可以有多个知识库吗？**

A: 当前版本每个租户一个知识库。多知识库/多 Bot 功能在"不做"列表中，后续根据用户反馈决定。

## API Key 相关（V1）

**Q: API Key 和旧版 widget_token 有什么区别？**

A: API Key（`dc_` 前缀）是 V1 新增的鉴权方式，支持命名、吊销、重命名，每个租户最多 5 个，通过 API Key 的对话计入用量统计。旧版 widget_token 是 MVP 的单一令牌，不支持统计和限额。建议新项目使用 API Key。

---

**Q: API Key 创建后还能查看完整值吗？**

A: 不能。API Key 仅在创建时显示一次完整值，之后只能看到脱敏后的部分（如 `dc_ab****xyz`）。请务必在创建时立即复制保存。

---

**Q: 每个租户可以创建多少个 API Key？**

A: 每个租户最多 5 个 API Key。如需更多，请先吊销不再使用的 Key。

---

**Q: 吊销 API Key 后可以恢复吗？**

A: 不可以。吊销操作不可撤销，使用该 Key 的聊天组件将立即无法对话。请确保已替换为新 Key 后再吊销旧 Key。

---

**Q: 对话返回"每日调用次数超限"怎么办？**

A: 这表示您的租户当日对话调用量已达到限额。请等待次日零点限额重置，或联系平台管理员调整限额。

---

**Q: 旧版嵌入脚本的 data-token 还能用吗？**

A: 过渡期内仍然可用，但建议尽快迁移到 `data-api-key`。使用 `data-token` 的对话不计入用量统计。

## 用量统计相关（V1）

**Q: 为什么用量统计中没有数据？**

A: 用量统计仅统计通过 API Key 鉴权的对话。如果您的聊天组件使用旧版 `data-token`，对话不会计入统计。请将嵌入脚本迁移到 `data-api-key`。

---

**Q: 预览对话会计入统计吗？**

A: 不会。管理后台的预览对话使用 JWT 鉴权，不计入用量统计，也不受限额约束。

---

**Q: 统计数据是实时的吗？**

A: 不是。统计数据存在短暂延迟（通常不超过 5 分钟），按自然日聚合。

## 评测集相关（V1）

**Q: Hit Rate 多少算合格？**

A: 没有统一标准，取决于业务场景。一般建议核心场景 Hit Rate > 80%。如果低于预期，建议优化文档结构或切分参数。

---

**Q: 评测集的问答对可以超过 50 对吗？**

A: 不可以。每个评测集最多 50 对问答。如需更多，可创建多个评测集。

---

**Q: 评测执行需要多长时间？**

A: 取决于问答对数量和 RAG 服务响应速度。通常 10 对问答在 30 秒内完成。

---

**Q: 评测对话会计入用量统计吗？**

A: 不会。评测过程中产生的对话不计入用量统计。

---

**Q: 期望文档名称必须完全匹配吗？**

A: 是的。期望文档名称需与知识库中的文档名称完全一致，包括文件扩展名。

## LLM 配置相关（V1）

**Q: 不配置 LLM 可以使用吗？**

A: 可以。未配置时系统使用默认的 LLM 服务（讯飞 Coding Plan API），开箱即用。

---

**Q: 支持哪些 LLM 供应商？**

A: 兼容 OpenAI Chat Completions API 格式的供应商均可使用，包括 OpenAI、Azure OpenAI、通义千问（兼容模式）、DeepSeek 等。

---

**Q: LLM 配置后对话报错怎么办？**

A: 排查步骤：
1. 检查 API 地址是否正确（需包含完整路径）
2. 检查 API 密钥是否有效
3. 检查模型名称是否与供应商文档一致
4. 使用"测试连通性"功能验证
5. 如无法解决，可删除配置恢复系统默认

---

**Q: 修改 LLM 配置会影响正在进行的对话吗？**

A: 不会。修改后仅影响新的对话请求，正在进行的对话使用原配置完成。

---

**Q: API 密钥安全吗？**

A: 密钥在界面和 API 响应中脱敏显示，数据库中加密存储。配置严格按租户隔离，仅管理员可操作。

## 部署问题

**Q: Docker Compose 启动失败，Milvus 健康检查不通过？**

A: Milvus 启动较慢（约 90 秒），健康检查有 `start_period: 90s` 配置。请耐心等待，可使用以下命令查看状态：

```bash
docker compose logs milvus
docker compose ps
```

---

**Q: 后端启动报数据库连接失败？**

A: 请确认：
1. PostgreSQL 服务已启动且健康检查通过
2. 数据库连接信息正确（默认：`localhost:5432`，数据库名 `docchat`，用户名 `docchat`）
3. 密码与 `POSTGRES_PASSWORD` 环境变量一致

---

**Q: 前端页面打开后是空白？**

A: 请确认：
1. 后端 API 服务已启动（`localhost:8080`）
2. 前端配置的 API 代理地址正确
3. 浏览器控制台是否有跨域或网络错误

---

**Q: 如何查看服务日志？**

A: 使用 Docker Compose 查看日志：

```bash
# 查看所有服务日志
docker compose logs

# 查看特定服务日志
docker compose logs server
docker compose logs web

# 实时跟踪日志
docker compose logs -f server
```

---

**Q: V1 数据库迁移失败？**

A: V1 新增了 Flyway 迁移脚本（V5-V7）。请确认：
1. Flyway 已启用（`spring.flyway.enabled=true`）
2. 迁移脚本位于 `classpath:db/migration/`
3. 数据库用户有 DDL 权限
4. 查看后端日志中的 Flyway 错误信息

## 故障排除

**Q: 文档上传后状态一直是"待处理"？**

A: 可能原因：
1. 后端异步任务处理器未正常启动 → 检查 Redis 连接
2. 任务队列积压 → 查看任务页面是否有大量 pending 任务
3. 后端服务异常 → 查看后端日志

---

**Q: 聊天组件没有出现在网站上？**

A: 排查步骤：
1. 确认嵌入脚本已正确粘贴到 HTML 的 `</body>` 之前
2. 检查 `data-api-key`（或 `data-token`）是否正确
3. 检查 `data-api-url` 是否指向正确的后端地址
4. 打开浏览器开发者工具，查看 Console 和 Network 是否有错误
5. 确认聊天组件处于 **启用** 状态

---

**Q: 对话返回"服务暂时不可用"？**

A: 可能原因：
1. LLM 服务不可用 → 检查 LLM 配置或系统默认 LLM 状态
2. Milvus 服务未启动 → 检查 Milvus 健康状态
3. 知识库中没有已完成的文档 → 确认至少有一个文档状态为"已完成"
4. API Key 无效或已吊销 → 检查 API Key 状态

---

**Q: 对话返回"API Key 无效或已吊销"？**

A: 排查步骤：
1. 确认嵌入脚本中的 `data-api-key` 值正确
2. 在 API Key 管理页面检查该 Key 的状态
3. 如果 Key 已吊销，创建新 Key 并更新嵌入脚本

---

**Q: 回答不准确或不相关？**

A: 改善建议：
1. 优化文档结构，使用清晰的标题层级
2. 确保文档内容完整，覆盖常见问题
3. 尝试调整切分参数（chunkSize 和 overlap）
4. 上传更专业的 FAQ 文档
5. 使用评测集功能量化评估检索质量，定位具体问题
6. 考虑更换更适合的 LLM 模型

---

**Q: LLM 连通性测试失败？**

A: 排查步骤：
1. 确认 API 地址格式正确，包含完整路径
2. 确认 API 密钥有效且有调用权限
3. 确认模型名称与供应商文档一致
4. 检查服务器网络是否能访问 LLM API 地址
5. 检查 LLM 供应商账户余额
