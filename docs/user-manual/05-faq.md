# 常见问题

## 通用问题

**Q: DocChat 支持哪些文档格式？**

A: 目前支持三种格式：PDF（.pdf）、Markdown（.md）和纯文本（.txt）。单文件大小限制为 50MB。

---

**Q: 上传文档后多久可以使用？**

A: 取决于文档大小和当前处理队列。通常小文档（< 1MB）在数秒内完成处理，大文档可能需要数分钟。您可以在任务页面查看处理进度。

---

**Q: 一个租户可以上传多少文档？**

A: MVP 阶段建议单个租户文档数量不超过 100 个，单文档不超过 50MB。如需更大容量，请联系我们。

---

**Q: 对话记录会保存吗？**

A: 不会。DocChat 遵循"对话即焚"原则，终端访客的对话不持久化存储，关闭聊天窗口后对话内容不可恢复。

---

**Q: 支持多轮对话吗？**

A: MVP 阶段不支持多轮上下文连续对话，每次提问独立处理。多轮对话将在后续版本中支持。

---

**Q: 可以有多个知识库吗？**

A: MVP 阶段每个租户一个知识库。多知识库/多 Bot 功能在"不做"列表中，后续根据用户反馈决定。

## 部署问题

**Q: Docker Compose 启动失败，Milvus 健康检查不通过？**

A: Milvus 启动较慢（约 90 秒），健康检查有 `start_period: 90s` 配置。请耐心等待，可使用以下命令查看状态：

```bash
docker compose logs milvus
docker compose ps
```

---

**Q: 后端启动报数据库连接失败？**

A: 请确认：
1. PostgreSQL 服务已启动且健康检查通过
2. 数据库连接信息正确（默认：`localhost:5432`，数据库名 `docchat`，用户名 `docchat`）
3. 密码与 `POSTGRES_PASSWORD` 环境变量一致

---

**Q: 前端页面打开后是空白？**

A: 请确认：
1. 后端 API 服务已启动（`localhost:8080`）
2. 前端配置的 API 代理地址正确
3. 浏览器控制台是否有跨域或网络错误

---

**Q: 如何查看服务日志？**

A: 使用 Docker Compose 查看日志：

```bash
# 查看所有服务日志
docker compose logs

# 查看特定服务日志
docker compose logs server
docker compose logs web

# 实时跟踪日志
docker compose logs -f server
```

## 故障排除

**Q: 文档上传后状态一直是"待处理"？**

A: 可能原因：
1. 后端异步任务处理器未正常启动 → 检查 Redis 连接
2. 任务队列积压 → 查看任务页面是否有大量 pending 任务
3. 后端服务异常 → 查看后端日志

---

**Q: 聊天组件没有出现在网站上？**

A: 排查步骤：
1. 确认嵌入脚本已正确粘贴到 HTML 的 `</body>` 之前
2. 检查 `data-token` 是否正确
3. 检查 `data-api-url` 是否指向正确的后端地址
4. 打开浏览器开发者工具，查看 Console 和 Network 是否有错误
5. 确认聊天组件处于 **启用** 状态

---

**Q: 对话返回"服务暂时不可用"？**

A: 可能原因：
1. 讯飞 API Key 未配置或已过期 → 检查 `XUNFEI_API_KEY` 环境变量
2. Milvus 服务未启动 → 检查 Milvus 健康状态
3. 知识库中没有已完成的文档 → 确认至少有一个文档状态为"已完成"

---

**Q: 回答不准确或不相关？**

A: 改善建议：
1. 优化文档结构，使用清晰的标题层级
2. 确保文档内容完整，覆盖常见问题
3. 尝试调整切分参数（chunkSize 和 overlap）
4. 上传更专业的 FAQ 文档
